# Stream 模块学习笔记

---

## 📌 模块简介

本模块系统学习 Java Stream API 的核心操作与使用方式，重点掌握流式数据处理思想，并理解其与传统循环写法之间的差异。

## 📚 本模块知识点

---

### ① filter —— 条件过滤

作用：筛选符合条件的数据

示例：

```
.filter(n -> n > 10)
```

特点：

* 返回新 Stream
* 不修改原数据
* 支持链式调用

---

### ② map —— 数据转换

作用：将元素转换为另一种形式

示例：

```
.map(String::toUpperCase)
```

本质：

```
T → R 转换
```

---

### ③ sorted —— 排序

作用：对流数据排序

方式：

默认排序：

```
.sorted()
```

自定义排序：

```
.sorted((a,b) -> b-a)
```

---

### ④ collect —— 结果收集

作用：将 Stream 转换为集合或结果对象

常见收集器：

| 方法      | 作用     |
| ------- | ------ |
| toList  | 转 List |
| toSet   | 转 Set  |
| toMap   | 转 Map  |
| joining | 拼接字符串  |

---

### ⑤ groupingBy —— 分组统计

作用：按规则对数据分组

示例：

```
Collectors.groupingBy(String::length)
```

返回：

```
Map<key,List>
```

---

### ⑥ Stream vs 传统循环

核心思想差异：

| 类型     | 思想      |
| ------ | ------- |
| for循环  | 告诉程序怎么做 |
| Stream | 告诉程序做什么 |

Stream 写法更清晰、更易维护。

---

## 🧠 Stream 执行模型（重点理解）

Stream 操作流程：

```
数据源 → 中间操作 → 终止操作 → 结果
```

示例流程：

```
list.stream()
    .filter(...)
    .map(...)
    .collect(...)
```

---

## ⚙ 操作类型分类

### 中间操作（不会执行）

* filter
* map
* sorted

特点：

```
惰性执行
```

---

### 终止操作（触发执行）

* forEach
* collect
* toList
* count

特点：

```
执行后流关闭
```
